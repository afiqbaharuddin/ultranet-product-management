Laravel Developer Assessment Test
Laravel Developer Assessment Test
Objective
The objective of this assessment is to evaluate the candidate's expertise in Laravel,
including backend development, API design, database management, security,
performance optimization, and deployment. The test is designed to assess problem
solving skills, code quality, and best practices.
Instructions
● You have 24 hours to complete this test.
● You must use Laravel (latest stable version) for backend development.
● Use MySQL as the database.
● Code must follow best practices and be structured using MVC architecture. .
The completed test must be submitted via GitHub with a README file
explaining your approach.
Task Description
You need to build a Product CRUD (Admin Dashboard) and Frontend where the
admin is able to manage products. RESTful API where it’s accessible by
authenticated admin. The system must include the following functionalities:
1. Product Management
CRUD operations (Admin Dashboard)
● Create a product (name, category_id, description, price, stock, enabled).
● Read products (with filtering by status and pagination support).
● Update product.
● Delete a product (with soft delete, and bulk delete features).
● Export products to excel
Eloquent Relationship
● Create a model and migration named Category (name),
● Establish a one-to-many relationship between Category and Product,
● Modify the json response to display category id and category name.
Restful API
● Create product
● Lists of products (Filter products based on the Category)
● Product detail
● Delete product
● Bulk delete products
2. Advanced Features
● Implement request validation using Laravel Form Requests.
● Implement unit tests for key functionalities using PHPUnit.
● Implement API documentation using Laravel Swagger or Laravel API resources.
● Implement packages support for SpartnerNL/Laravel-Excel
Technical Requirements
● Laravel (latest stable version)
● MySQL for database
● Laravel Form Requests for validation
● PHPUnit for testing
Evaluation Criteria
Candidates will be evaluated based on the following criteria:
● Functionality – Does the application meet all requirements?
● Code Quality – Is the code well-structured, readable, and maintainable?
● Security – Is authentication and authorization implemented properly?
● Performance – Is the API optimized for performance and scalability?
● Testing – Are unit tests included and properly structured?
● Documentation – Is the README file and Swagger documentation clear
and complete?
Submission Guidelines
● Push your code to a public GitHub repository.
● Include a README.md file explaining:
○ Project setup instructions
○ API endpoints documentation
○ Assumptions and design choices
● Share the repository link before the deadline.
Bonus Points
● Implement Docker support for easy deployment.
Good Luck!
